# BUG 修复记录 - 2026-06-16

## 修复的高严重程度 BUG (4个)

### BUG-001: TrafficHub 连接断开时未清理订阅关系
- **文件**: `Infrastructure/Realtime/TrafficHub.cs`
- **问题**: 连接断开时只移除连接 ID，不清理空的订阅条目，导致内存泄漏
- **修复**: 连接断开后检查 HashSet 是否为空，为空则移除整个路由条目

### BUG-002: LogStreamController 自定义 Channel 实现错误
- **文件**: `Modules/Dashboard/Controllers/LogStreamController.cs`
- **问题**: 自定义的 Channel 实现使用 BlockingCollection，`WaitToReadAsync` 同步返回结果而非真正的异步等待
- **修复**: 删除自定义实现，改用标准 `System.Threading.Channels`

### BUG-003: RequestRetryMiddleware 重试时请求体问题
- **文件**: `Modules/Retry/Middleware/RequestRetryMiddleware.cs`
- **问题**: 重试时尝试 seek 请求体，但流式请求体（如 PushStreamContent）无法 seek，导致重试失败
- **修复**: 首次读取后用 `MemoryStream` 重新填充请求体，而非依赖 `Seek`

### BUG-004: DashboardServiceCollectionExtensions ServiceProvider 泄漏
- **文件**:
  - `Extensions/DashboardServiceCollectionExtensions.cs`
  - `Extensions/StorageServiceCollectionExtensions.cs`
- **问题**: `AddAneiangStorage()` 内部调用 `services.BuildServiceProvider()` 创建的 ServiceProvider 未释放，导致资源泄漏
- **修复**: 移除 `IConfiguration` 参数传递，改为通过配置绑定方式获取
