# Aneiang.Yarp 企业级开源治理路线图

> 目标：围绕身份一致性、存储迁移、性能、安全、项目结构、配置部署六大方向，把 Aneiang.Yarp 从功能型网关平台提升为企业级开源项目。  
> 原则：先止血、再过渡、再正式化；所有改造必须兼容现有 API 和现有 SQLite 数据。

---

# 1. 身份与一致性必须项

## 1.1 目标

建立稳定的实体身份模型：

```text
UID：内部不可变身份
Key：YARP 对外 ID，可受控重命名
DisplayName：用户展示名，可自由修改
```

当前过渡字段：

```text
RouteUid + RouteId
ClusterUid + ClusterId
PolicyUid + PolicyId
```

最终命名建议：

```text
RouteUid + RouteKey + DisplayName
ClusterUid + ClusterKey + DisplayName
PolicyUid + PolicyKey + DisplayName
```

- [x] 普通编辑禁改 `routeId` / `clusterId`
- [x] 后端 Save API 拒绝通过普通保存改 ID

- [x] `GatewayIdentityService` 统一处理 rename 后副作用
- [x] `RouteEntity` / `ClusterEntity` / `PolicyEntity` 增加 UID
