# 策略系统重构设计文档

## 一、现状问题

### 1.1 策略是"死"的

当前策略管理页面只做 CRUD，创建出来的策略模板无法被任何路由或集群使用。
`GatewayPolicyService.ApplyPolicyToRouteAsync()` 方法虽已实现，但：

- 未注册到 DI 容器
- 没有 API 端点暴露
- 前端没有调用入口
- 策略和路由/集群之间完全断裂

### 1.2 熔断器层级错位

- **配置来源**：从路由 `Metadata` 读取（路由级）
- **运行时状态**：按 `{clusterId}:{destinationId}` 跟踪（集群级）

同一个集群下的多个路由可能配置不同的熔断参数，但共享同一个熔断状态 → 逻辑矛盾。

### 1.3 WAF 两套体系冲突

- `GatewayPolicy` 模型中有 `PolicyWaf`，其 `ToMetadata()` 输出 `Waf:Enabled` 等键
- `WafMiddleware` 实际从全局 `WafOptions` 读取配置，不读取路由元数据
- 策略里的 WAF 开关形同虚设

### 1.4 概念混杂

一个 `GatewayPolicy` 同时包含四种不相关的能力（熔断、重试、限流、WAF），
但它们实际上分属不同层级（全局 / 集群 / 路由），用户难以理解"一个策略到底管什么"。

---

## 二、设计原则

1. **每个特性放在它最自然的层级** — 熔断天生是集群级，限流/重试天生是路由级，WAF 是全局+路由开关
2. **策略模板是"一键应用"的工具，不是孤立的配置** — 创建策略之后必须有明确的"应用到哪"的流程
3. **用户一眼就能看清** — 策略页看到"应用到哪些路由/集群"，路由页看到"绑了什么策略"，集群页看到"熔断怎么配的"
4. **不做过度设计** — 不引入复杂的策略继承/优先级/合并机制

## 三、新架构

### 3.1 分层归属

| 功能 | 归属层级 | 配置存储位置 | 状态查看 |
|------|---------|------------|---------|
| **熔断器** | 集群级 | `DynamicClusterConfig.CircuitBreaker` | `/circuits` |
| **重试** | 路由级 | `RouteConfig.Metadata` → `Retry:*` | `/logs` |
| **限流** | 路由级 | `RouteConfig.Metadata` → `RateLimit:*` | `/logs`（429响应） |
| **WAF 全局开关** | 全局级 | `WafOptions`（appsettings / 设置页） | `/security` |
| **WAF 路由开关** | 路由级 | `RouteConfig.Metadata` → `Waf:Enabled` | `/security` |

### 3.2 策略模板重组

```
路由策略模板 (RoutePolicy)
├── 重试 (Retry)       → 应用时写入目标路由的 Metadata
├── 限流 (Rate Limit)  → 应用时写入目标路由的 Metadata
└── WAF 开关           → 应用时写入目标路由的 Metadata（覆盖全局默认）

集群策略模板 (ClusterPolicy)  ← 新增类别
└── 熔断器 (Circuit Breaker) → 应用时写入目标集群的 CircuitBreaker 配置
```

### 3.3 数据流

```
┌─────────────────────────────────────────────────────┐
│  路由策略模板                                         │
│  [重试 + 限流 + WAF开关]                              │
│         │                                            │
│         ▼ "应用到路由"                                │
│   RouteConfig.Metadata                               │
│   ├── Retry:Enabled, Retry:MaxRetries ...            │
│   ├── RateLimit:Enabled, RateLimit:PermitLimit ...   │
│   └── Waf:Enabled = true/false                       │
│         │                                            │
│         ▼ 中间件读取                                   │
│   RequestRetryMiddleware ← Retry:*                   │
│   RateLimitMiddleware   ← RateLimit:*                │
│   WafMiddleware         ← Waf:Enabled (新增)         │
└─────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────┐
│  集群策略模板                                         │
│  [熔断器]                                             │
│         │                                            │
│         ▼ "应用到集群"                                │
│   DynamicClusterConfig.CircuitBreaker                │
│   ├── Enabled, FailureThreshold                      │
│   ├── RecoveryTimeoutSeconds, HalfOpenMaxAttempts    │
│   └── FailureStatusCodes                             │
│         │                                            │
│         ▼ 中间件读取                                   │
│   CircuitBreakerMiddleware ← 从集群配置读取（不再从路由元数据）│
└─────────────────────────────────────────────────────┘
```

## 四、模型变更

### 4.1 删除

| 类型 | 原因 |
|------|------|
| `GatewayPolicyCollection` | 不再需要集合包装器 |
| `GatewayPolicyPersistenceService` 及接口 | 策略改为通过 IGatewayRepository 存储，不需要独立持久化 |

### 4.2 重构

#### GatewayPolicy → 拆分为 RoutePolicy + ClusterPolicy

```csharp
/// <summary>
/// 路由策略模板：包含重试、限流、WAF开关，可应用到路由。
/// </summary>
public class RoutePolicy
{
    public string PolicyId { get; set; } = string.Empty;
    public string DisplayName { get; set; } = string.Empty;
    public string? Description { get; set; }
    public bool Enabled { get; set; } = true;
    public DateTime CreatedAt { get; set; } = DateTime.UtcNow;

    /// <summary>绑定的路由ID列表（只读，由系统维护）。</summary>
    public List<string> AppliedRoutes { get; set; } = new();

    public PolicyRetry? Retry { get; set; }
    public PolicyRateLimit? RateLimit { get; set; }

    /// <summary>WAF路由级开关：true=强制开启，false=强制关闭，null=跟随全局。</summary>
    public bool? WafEnabled { get; set; }
}

/// <summary>
/// 集群策略模板：包含熔断器配置，可应用到集群。
/// </summary>
public class ClusterPolicy
{
    public string PolicyId { get; set; } = string.Empty;
    public string DisplayName { get; set; } = string.Empty;
    public string? Description { get; set; }
    public bool Enabled { get; set; } = true;
    public DateTime CreatedAt { get; set; } = DateTime.UtcNow;

    /// <summary>绑定的集群ID列表（只读，由系统维护）。</summary>
    public List<string> AppliedClusters { get; set; } = new();

    public PolicyCircuitBreaker? CircuitBreaker { get; set; }
}
```

#### DynamicClusterConfig 增加 CircuitBreaker

```csharp
public class DynamicClusterConfig
{
    // ... 现有字段保持不变 ...
    
    /// <summary>集群级熔断器配置。</summary>
    public CircuitBreakerConfig? CircuitBreaker { get; set; }
}

/// <summary>
/// 集群级熔断器配置。
/// </summary>
public class CircuitBreakerConfig
{
    public bool Enabled { get; set; } = true;
    public int FailureThreshold { get; set; } = 5;
    public int RecoveryTimeoutSeconds { get; set; } = 30;
    public int HalfOpenMaxAttempts { get; set; } = 1;
    public List<int> FailureStatusCodes { get; set; } = new() { 500, 502, 503, 504 };
}
```

### 4.3 保留不变

| 类型 | 原因 |
|------|------|
| `PolicyRetry` | 仅微调 `ToMetadata()` |
| `PolicyRateLimit` | 仅微调 `ToMetadata()` |
| `PolicyCircuitBreaker` | 保留模型定义，但从路由级剥离出来归入集群级 |
| `PolicyWaf` | **删除**，改为 RoutePolicy 上的 `bool? WafEnabled` 开关 |
| `PolicyEntity` | 扩展字段适配新模型 |

### 4.4 PolicyEntity 扩展

```csharp
public class PolicyEntity
{
    public string PolicyId { get; set; } = string.Empty;
    public string PolicyType { get; set; } = "route";   // "route" | "cluster"
    public string DisplayName { get; set; } = string.Empty;
    public string? Description { get; set; }
    public bool Enabled { get; set; } = true;
    public string? RetryConfig { get; set; }            // JSON
    public string? RateLimitConfig { get; set; }        // JSON
    public string? CircuitBreakerConfig { get; set; }   // JSON
    public string? WafEnabled { get; set; }             // "true" / "false" / null
    public string? AppliedTargets { get; set; }         // JSON: ["route-id-1", "route-id-2"] 或 ["cluster-id-1"]
    public DateTime CreatedAt { get; set; }
}
```

## 五、服务层变更

### 5.1 删除 GatewayPolicyPersistenceService

策略的存储统一走 `IGatewayRepository`，不再有两个持久化通道。

### 5.2 重构 GatewayPolicyService

```csharp
public interface IGatewayPolicyService
{
    // ─── 路由策略 ─────────────────────────────
    Task<IReadOnlyList<RoutePolicy>> GetAllRoutePoliciesAsync();
    Task<RoutePolicy?> GetRoutePolicyAsync(string policyId);
    Task<RoutePolicy> CreateRoutePolicyAsync(RoutePolicy policy);
    Task<RoutePolicy?> UpdateRoutePolicyAsync(string policyId, RoutePolicy policy);
    Task<bool> DeleteRoutePolicyAsync(string policyId);

    /// <summary>将路由策略应用到指定路由（写入路由 Metadata）。</summary>
    Task<bool> ApplyRoutePolicyAsync(string policyId, string routeId);
    /// <summary>从指定路由解绑路由策略（清除相关 Metadata）。</summary>
    Task<bool> UnapplyRoutePolicyAsync(string policyId, string routeId);

    // ─── 集群策略 ─────────────────────────────
    Task<IReadOnlyList<ClusterPolicy>> GetAllClusterPoliciesAsync();
    Task<ClusterPolicy?> GetClusterPolicyAsync(string policyId);
    Task<ClusterPolicy> CreateClusterPolicyAsync(ClusterPolicy policy);
    Task<ClusterPolicy?> UpdateClusterPolicyAsync(string policyId, ClusterPolicy policy);
    Task<bool> DeleteClusterPolicyAsync(string policyId);

    /// <summary>将集群策略应用到指定集群（写入集群 CircuitBreaker 配置）。</summary>
    Task<bool> ApplyClusterPolicyAsync(string policyId, string clusterId);
    /// <summary>从指定集群解绑集群策略。</summary>
    Task<bool> UnapplyClusterPolicyAsync(string policyId, string clusterId);
}
```

### 5.3 DynamicYarpConfigService 扩展

```csharp
// 新增方法
Task<bool> UpdateClusterCircuitBreakerAsync(string clusterId, CircuitBreakerConfig? config);
```

此方法更新集群内存配置中的熔断配置，并持久化到集群存储。

### 5.4 WafMiddleware 改造

现有逻辑：从 `WafOptions` 全局读取配置。
改造后：

```
1. 检查当前路由是否有 Metadata["Waf:Enabled"]
   - "true"  → 强制开启 WAF
   - "false" → 强制关闭 WAF（即使全局开着也跳过）
   - 不存在  → 回退到 WafOptions 全局默认
2. 如果决定开启，按 WafOptions 中的规则检测
3. 阻断的事件照常写入 WafEventStore
```

## 九、关键决策记录

| 决策 | 理由 |
|------|------|
| 保留策略模板概念，而非让用户直接在路由/集群上配 | 模板可复用，避免 50 个路由重复配同样的重试参数 |
| 熔断器迁移到集群配置，而非给集群加 Metadata | 熔断是集群的固有属性，与负载均衡、健康检查并列更自然 |
| WAF 的 `WafEnabled` 用三态（true/false/null）| 既支持全局统一管控，又支持特殊路由单独开关 |
| 策略和应用关系是"快照写入"而非"实时引用" | 应用时将策略参数写入 Metadata / 集群配置，之后策略修改不影响已绑定目标。简化心智模型，避免级联影响 |
| AppliedTargets 存在 PolicyEntity 上 | 策略知道自己绑了谁，方便 UI 展示"已应用 N 个路由"；路由/集群侧通过 Metadata/Config 反向感知 |
