# 流量策略管理 — 使用指南

## 概述

「流量策略管理」是网关仪表盘中的策略模板系统，支持两种策略类型：

- **路由策略**：包含重试、限流、WAF 开关，应用到路由
- **集群策略**：包含熔断器配置，应用到集群

每个策略创建后可通过「应用」功能一键绑定到目标路由/集群，绑定后策略参数会写入目标配置中即时生效。

> **默认 Dashboard 访问地址**：`http://localhost:5000/apigateway/`
> *（具体端口和路由前缀请以你的项目配置为准）*

---

## 一、策略入口

登录 Dashboard → 左侧导航栏 **「安全策略」** → **「策略管理」**（`/apigateway/policies`）。

页面顶部有两个 Tab：

| Tab | 内容 |
|-----|------|
| **路由策略** | 管理重试 + 限流 + WAF 开关的策略模板 |
| **集群策略** | 管理熔断器配置的策略模板 |

---

## 二、路由策略

### 2.1 功能模块

| 功能 | 说明 |
|------|------|
| **重试** | 请求失败时自动重试，支持指数退避和随机抖动 |
| **限流** | 限制单位时间窗口内的请求数量，三种算法可选 |
| **WAF 开关** | 路由级覆盖全局 WAF 设置：强制开启 / 强制关闭 / 跟随全局 |

### 2.2 创建路由策略

1. 切换到「路由策略」Tab
2. 点击右上角 **「+ 新增策略」**
3. 填写基础信息

| 字段 | 必填 | 说明 | 示例 |
|------|------|------|------|
| 策略ID | 是 | 唯一标识，创建后不可修改 | `high-availability` |
| 显示名称 | 否 | 便于识别 | `高可用策略` |
| 描述 | 否 | 用途说明 | `适用于核心业务路由` |

4. 按需启用功能模块：

#### 重试参数

| 参数 | 默认值 | 说明 |
|------|--------|------|
| 最大重试次数 | 3 | 失败后最多重试 N 次 |
| 退避基数（ms） | 100 | 实际延迟 = 基数 × 2^重试次数 + 随机抖动 |

#### 限流参数

| 参数 | 默认值 | 说明 |
|------|--------|------|
| 限制数 | 100 | 每窗口允许的最大请求数 |
| 时间窗口 | 1m | 窗口大小（`1s` / `1m` / `1h`） |
| 算法 | FixedWindow | FixedWindow / SlidingWindow / TokenBucket |

#### WAF 参数

| 选项 | 含义 |
|------|------|
| ● 强制开启 | 即使全局 WAF 关闭，该路由也启用 WAF 检测 |
| ● 强制关闭 | 即使全局 WAF 开启，该路由跳过 WAF 检测 |
| ● 跟随全局默认 | 不写入覆盖，跟随 `/settings` 中 WAF 总开关 |

5. 点击 **「创建策略」**

### 2.3 应用路由策略到路由

1. 在路由策略列表中，点击目标策略行的 **🔗 应用** 按钮
2. 弹窗中列出所有路由，勾选要应用的路由
3. 已勾选 = 已绑定，取消勾选 = 解绑
4. 点击 **「确认」**，策略参数立即写入目标路由的 Metadata

> **应用是快照写入**：应用时将策略参数写入路由配置，之后修改策略模板不会自动同步已绑定的路由。如需更新，请重新应用。

## 三、集群策略

### 3.1 功能模块

| 功能 | 说明 |
|------|------|
| **熔断器** | 当后端服务连续失败达到阈值时自动熔断，停止转发请求到该集群 |

> **为什么熔断在集群级别？** 熔断保护的是下游服务健康，而不是某个入口路径。同一个集群下的所有路由共享同一份熔断状态。

### 3.2 创建集群策略

1. 切换到「集群策略」Tab
2. 点击右上角 **「+ 新增策略」**
3. 填写基础信息（策略ID、显示名称、描述）
4. 配置熔断器参数：

| 参数 | 默认值 | 说明 |
|------|--------|------|
| 失败阈值 | 5 | 连续失败 N 次后熔断 |
| 恢复超时（秒） | 30 | 熔断后等待 N 秒再尝试恢复 |
| 半开最大请求 | 1 | 半开状态下允许通过的试探请求数 |

5. 点击 **「创建策略」**

### 3.3 应用集群策略到集群

1. 在集群策略列表中，点击目标策略行的 **🔗 应用** 按钮
2. 弹窗中列出所有集群，勾选要应用的集群
3. 点击 **「确认」**，熔断配置立即写入目标集群

## 四、查看策略生效结果

### 4.1 熔断状态 — 实时监控

**路径**：`/apigateway/circuits`（导航：「监控运维」→「熔断状态」）

| 状态 | 含义 |
|------|------|
| 🟢 Closed | 正常转发，未达失败阈值 |
| 🔴 Open | 已熔断，请求直接返回 503 |
| 🟡 HalfOpen | 试探恢复中 |

每行显示：集群名、目标地址、状态、失败次数/阈值、恢复倒计时。

### 4.2 日志 — 请求级追踪

**路径**：`/apigateway/logs`（导航：「监控运维」→「日志」）

| 场景 | 日志特征 |
|------|----------|
| 限流触发 | 状态码 429 的 ProxyResponse |
| 熔断触发 | 状态码 503 的 ProxyResponse |
| 重试执行 | 同一 TraceId 出现多次 ProxyRequest/ProxyResponse |
| WAF 阻断 | 状态码 403，Message 含 `waf: true` |

### 4.3 安全事件 — WAF 攻击拦截

**路径**：`/apigateway/security`（导航：「安全策略」→「安全事件」）

展示所有被 WAF 拦截的请求，按攻击类型分布和 Top 攻击 IP 排行。

### 4.4 访问统计 — 整体流量

**路径**：`/apigateway/stats`（导航：「监控运维」→「访问统计」）

查看成功率、延迟、QPS、状态码分布。

### 4.5 集群页面 — 确认熔断配置

**路径**：`/apigateway/clusters`（导航：「网关管理」→「服务集群」）

点击集群展开详情，可看到熔断器区块显示绑定的策略名称和参数。

### 4.6 路由页面 — 确认策略 Metadata

**路径**：`/apigateway/routes`（导航：「网关管理」→「路由配置」）

点击路由展开详情，在「路由策略」区块可看到绑定的策略名称和已写入的参数。

## 五、全局 WAF 配置

**路径**：`/apigateway/settings`（导航：「系统管理」→「设置」）

WAF 总开关和全局规则在此配置。路由策略中的 WAF 开关可以覆盖全局设置：

| 全局 WAF | 路由 WAF 开关 | 实际效果 |
|----------|--------------|---------|
| 开 | 跟随全局（默认） | ✅ WAF 生效 |
| 开 | 强制关闭 | ❌ 该路由跳过 WAF |
| 关 | 强制开启 | ✅ 该路由仍启用 WAF |
| 关 | 跟随全局（默认） | ❌ WAF 不生效 |

## 六、完整操作流程

```
Step 1: 创建路由策略
  路由策略 Tab → 新增策略 → 填写重试/限流/WAF参数 → 创建

Step 2: 应用到路由
  点击策略行的 🔗 应用按钮 → 勾选路由 → 确认

Step 3: 创建集群策略
  集群策略 Tab → 新增策略 → 填写熔断参数 → 创建

Step 4: 应用到集群
  点击策略行的 🔗 应用按钮 → 勾选集群 → 确认

Step 5: 观察效果
  /circuits   → 看熔断器状态
  /logs       → 看请求日志（429/503/重试）
  /security   → 看 WAF 拦截记录
  /stats      → 看整体流量统计
```

## 七、常见问题

### Q1：策略应用后修改模板，已绑定的路由/集群会自动更新吗？

不会。应用是快照写入，策略参数写入目标配置后即独立。修改模板后需重新应用才能生效。这是为了避免级联影响。

### Q2：一个路由可以绑定多个路由策略吗？

可以，但后应用的策略会覆盖相同 Key 的 Metadata 条目（如 `Retry:MaxRetries`）。建议每个路由只绑定一个策略。

### Q3：限流的全局和路由级有什么区别？

路由策略中的限流写入路由 Metadata，只对该路由生效。`/settings` 中的限流是 ASP.NET Core 全局限流器，保护整个 Dashboard。

### Q4：熔断器状态在哪里重置？

`/circuits` 页面提供「全部重置」按钮，将所有断路器恢复到 Closed 态。

### Q5：路由策略中的 WAF「强制关闭」有什么用？

某些内部 API（如健康检查、监控回调）不需要 WAF 检测但又经过网关，可以用「强制关闭」跳过 WAF 避免误拦截。
